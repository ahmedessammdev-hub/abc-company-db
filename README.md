## Session 1: HR & Employee Management System

### Database Schema
* **Departments**: Department lookup (DepartmentID, DepartmentName).
* **Employees**: Employee registry containing hire date, supervisor details (self-referencing manager ID), and status.
* **Salaries**: Stores salary rates and pay structures (Monthly or Annual rate).
* **Projects**: Tracks active and archived company projects.
* **ProjectAssignments**: Junction table mapping employees to projects with logged project hours.

### Tasks & Query Solutions

* **Task 1: Employee & Salary List**: Joins employees with their departments and salaries using `LEFT JOIN` to ensure employees without active salary records are listed.
* **Task 2: Department Headcount**: Aggregates employee count grouped by department, including departments with zero employees.
* **Task 3: Unassigned Employees**: Lists active employees who have not been assigned to any project.
* **Task 4: Department Salary Summary**: Displays the sum, average, and headcount of salary expenditures per department, sorted by highest spend.
* **Task 5: Employee & Manager Report**: Utilizes a self-join on the `Employees` table to display employees side-by-side with their direct managers.
* **Task 6: Active Projects with High Participation**: Identifies active projects staffed by more than 3 distinct employees and showing high logged hours.
* **Task 7: Department–Project Participation Matrix**: Generates a matrix matching every department to every project (via `CROSS JOIN`) to report employee assignment metrics.
* **Task 8: Scalar Function (Employee Tenure)**: Implements `fn_get_emp_tenure` to calculate how many full calendar years an employee has worked based on their hire date.
* **Task 9: Scalar Function (Annual Salary)**: Implements `fn_annual_salary` to compute annual salary rates based on Monthly vs. Annual pay structures.
* **Task 10: Table-Valued Function (Department Employee List)**: Implements `fn_dept_employees` (Inline TVF) returning employee details for a target department, queried using `CROSS APPLY`.
* **Task 11: Stored Procedure (Department Salary Report)**: Implements `sp_dept_salary_report` returning employee listings and computing aggregates (count, sum, average, top earner) via `OUTPUT` variables.
* **Task 12: Stored Procedure (Give Department Raise)**: Implements `sp_give_raise` updating active salary rates inside a department by a target percentage, secured inside a transaction block.

## Session 2: Advanced E-Commerce Order Management System

### Database Schema
* **Categories**: Hierarchical category tree with recursive self-links (ParentCategoryID).
* **Customers**: User accounts registry capturing email profiles and registration times.
* **Products**: Inventory catalogs tracking prices, category associations, and quantities in stock.
* **Orders**: Purchase invoices header tracking order date and order state (Pending, Completed, Cancelled).
* **OrderItems**: Purchase line items junction table capturing exact quantity and frozen price paid per item.
* **Payments**: Financial records tracking payment dates, amounts, and states (Pending, Paid, Failed) per order.
* **RestockAlerts**: Target table populated by automated inventory cursor alerts.
* **MonthlySalesAudit**: Archive table logging monthly sales aggregations generated by audit routines.

### Tasks & Query Solutions

* **Task 1: Customer Order & Spend List**: Generates customer lists with total order counts and lifetime expenditures, utilizing `LEFT JOIN` to retain non-active customers.
* **Task 2: Active Customers with Unpaid Orders**: Lists customers with > 3 orders and >= 1 unpaid invoice using a nested subquery to handle order-payment validations.
* **Task 3: Product Performance Report**: Identifies unique customer counts, total units sold, and total revenues generated per product, ordered by total revenue.
* **Task 4: Order Payment Processing Time**: Computes the day gap between order placements and successful payments, returning `NULL` for unpaid orders.
* **Task 5: Top 3 Best-Selling Products per Category**: Extracts the top 3 best-selling products per category using `ROW_NUMBER() OVER (PARTITION BY CategoryID ORDER BY SUM(Quantity) DESC)`.
* **Task 6: Customer Spending Tier Classification**: Ranks customer spending tiers (Above Average, Below Average, Average) against the overall spending average of all customers.
* **Task 7: Monthly Revenue and Cumulative Growth**: Reports monthly revenue aggregates and calculates running cumulative revenue totals over the past 12 months.
* **Task 8: Quick Action Customers (First order within 7 days)**: Queries customers whose earliest order date fell within 7 days of their registration date.
* **Task 9: Recursive Category Hierarchy Builder**: Resolves full nested paths for category breadcrumbs (e.g. `Electronics > Mobiles > Smartphones`) recursively.
* **Task 10: Customer Order Summary View**: Defines view `vw_customer_order_summary` capturing customer order metrics and queries the top 10 spenders from it.
* **Task 11: Covering Index Optimization**: Creates non-clustered index `IX_OrderItems_ProductID_Covering` to switch the execution plan of Task 3 from table scan to index seek.
* **Task 12: Customer Lifetime Value Function**: Implements scalar function `fn_customer_lifetime_value` to calculate total paid invoice amounts for a customer.
* **Task 13: Order Value Discount Function**: Implements scalar function `fn_order_discount` to calculate tiered discounts (10% over 10k, 5% over 5k) on order subtotals.
* **Task 14: Orders by Date Range TVF**: Implements Table-Valued Function `fn_orders_by_date_range` to pull order details, tested using `CROSS APPLY` to generate monthly snapshots.
* **Task 15: Stored Procedure - Place Order (Transaction Safe)**: Implements `sp_place_order` executing order insertions, inventory deduction, and payment records inside a safe database transaction block with automatic `ROLLBACK` on failure.
* **Task 16: Monthly Sales Report SP**: Implements `sp_monthly_sales_report` returning overall metrics via output variables, and outputting top products and customers as result sets.
* **Task 17: Low Stock Alerts Cursor**: Populates restock alerts using a procedural loop checking products with stock quantity below 10.
* **Task 18: Monthly Sales Audit Cursor**: Audits and archives totals for the last 12 months in `MonthlySalesAudit` by executing `sp_monthly_sales_report` iteratively via a cursor loop.

## How to Execute the Scripts

1. Connect to your Microsoft SQL Server instance using SQL Server Management Studio (SSMS) or Azure Data Studio.
2. Navigate to the target folder (`Session 1` or `Session 2`).
3. Open and execute the scripts in the specified order:
   - Run `01_create_tables.sql` to build the database database and tables.
   - Run `02_insert_data.sql` to seed the database with mock records.
   - Run `03_task_queries.sql` to run the task queries, define routines, and run verification routines.
